# Frontend Rules

- Server state must use React Query
- Client global state should use Zustand only when necessary
- Avoid unnecessary global state
- Avoid prop drilling by proper component design
- API logic must be separated from UI
- Components should remain focused and small
- Avoid large page components
- Prefer custom hooks for reusable logic
- Avoid useEffect for simple data fetching
- Prefer composition over complex inheritance patterns
- Use TypeScript strictly
- Avoid any
- Keep business logic outside JSX
